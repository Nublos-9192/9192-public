HTTP API + sandbox + receipt

Run your first verified machine call in 5 minutes.

Start with the public HTTP API, execute a free sandbox call, and verify the receipt before touching funding or paid execution.

Run shortest testRead full flowCheck status
1

Check status

Confirm the public gateway is online.

curl https://nineoneninetwo.com.br/api/v1/status

Expected: "status":"OK"

2

Run smoke

Read status, pricebook, payment info, HELLO, CAPS and free quotes without spending balance.

npx @nineoneninetwo/demo --mode smoke

Expected: discovery, status and a free quote response.

3

Run sandbox

Execute the first GET_PULSE path in the free limited sandbox.

npx @nineoneninetwo/demo --mode sandbox

Expected: quote, execution and receipt verification complete.

4

Create quote

Create a free limited quote directly through JSON when you are ready to integrate.

POST /api/v1/sandbox/quotes/get-pulse

Use a small machine id, bits=65536, and max_output_bytes=8192.

5

Execute test call

Accept the sandbox quote and execute the test call.

POST /api/v1/sandbox/quotes/{quote_id}/accept
POST /api/v1/sandbox/executions/get-pulse

Expected: a receipt id or receipt payload from the sandbox response.

6

Verify receipt

Close the loop by verifying the receipt for free.

POST /api/v1/sandbox/receipts/verify

Expected: verification accepted by the public API.

7

Go paid

Only after the verified sandbox path works, move to real execution with invoice funding and crypto settlement.

python examples/9192_public_api_client.py invoice --machine first_client --units 100000 --network TRON --asset USDT
